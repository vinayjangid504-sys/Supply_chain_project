# Supply_chain_project
# 📦 Supply Chain Management System (SQL Project)

## 📌 Overview

This project is based on a **Supply Chain Management System** built using MySQL.
It demonstrates how a relational database can be used to manage customers, orders, products, and suppliers efficiently.

The goal of this project is to understand real-world database design and solve business problems using SQL queries.

---

## 🎯 Objective

* Design a relational database schema
* Implement tables using SQL (DDL)
* Apply constraints like Primary Key & Foreign Key
* Perform data manipulation (DML)
* Solve real-world queries using SQL

---

## 🗂️ Database Structure

This project contains **5 main tables**:

* **Customer** – Stores customer details
* **Orders** – Stores order information
* **OrderItem** – Stores product details within each order
* **Product** – Stores product data
* **Supplier** – Stores supplier information

---

## 🔗 Relationships

* One Customer ➝ Many Orders
* One Order ➝ Many OrderItems
* One Product ➝ Many OrderItems
* One Supplier ➝ Many Products

---

## ⚙️ SQL Concepts Used

* Joins (INNER JOIN)
* Aggregation (SUM, COUNT)
* Subqueries
* CTE (Common Table Expressions)
* Window Functions
* Group By & Order By

---

## 💡 Key Features

* Track customer orders
* Analyze product demand
* Identify top suppliers
* Calculate revenue and order trends
* Perform real-world business analysis

---

## 📊 Sample Insights (Queries)

* Country-wise customer count
* Year-wise total revenue
* Top 3 suppliers by revenue
* Customers who didn’t place any orders
* Most demanded products
